服务器端网络编程的四个步骤：

  • 调用 socket 函数创建套接字
  • 调用 bind 函数分配 IP 地址和端口号（本地）
  • 调用 listen 函数将套接字转为可接受请求状态
  • 调用 accept 函数受理连接请求

客户端网络编程的两个步骤：

  • 调用 socket 函数创建套接字
  • 使用 connect 函数请求连接到服务器

在 linux 的世界里，socket 也被认为是文件中的一种，所以在网络数据传输的过程中也可以使用文件 IO 相关的函数。

#include <sys/socket.h>

// 成功时返回文件描述符，失败时返回-1
int socket(int domain, int type, int protocol);

Domian 指定协议族信息，常用的是 PF_INET 和 PF_INET6。type 指的是套接字类型，指定了套接字的数据传输方式，一般分为面向连接的套接字 SOCK_STREAM 和面向消息的套接字 SOCK_DGRAM。protocol 参数决定最终采用的协议。

大部分情况下，使用第一个参数和第二个参数即可以指定所需协议，所以大多数情况下可以将第三个参数设置为 0，除非遇到“同一协议族中存在多个数据传输方式相同的协议”。需要注意，第一个参数指定的是协 议族，第二个参数指定的是传输方式，有的协议族中，凭借这两个参数确实无法唯一指定一个协议。

sock_t sock = socket(PF_INET, SOCK_STREAM,0);   // 意指 IPV4 协议族中面向连接的套接字，当前此协议族只有 TCP 符合这个要求，所以第三个参数可以传递为 0.

面向连接的套接字

面向连接的套接字的特点有：

  • 输出传输过程中不会消失
  • 按序传输数据
  • 传输的数据不存在数据边界

100 个糖果是分批传递的，但是接受者凑齐 100个后才装袋。

传输数据的计算机通过 3 次调用 write 函数传递了 100 字节的数据，但接受数据的计算机仅通过一次 read 函数调用就接收到了全部 100 字节。

收发数据的套接字内部有缓冲。收到数据并不意味着马上调用 read 函数。只要不超过缓冲容量，则有可能在数据填充满缓冲后通过 1 次 read 函数调用读取全部，也有可能分成多次 read 函数调用进行读取。也就是说，在面向连接的套接字中，read 函数和 write 函数的调用次数并无太大的意义。

在面向连接的套接字中，即便接收端的缓冲满了，也不会发生数据丢失的现象。此外，套接字连接必须一一对应。一句话概括面向连接的套接字就是：

可靠的、有序的、基于字节的面向连接的数据传输方式的套接字。

面向消息的套接字

面向消息的套接字的特点是：

  • 强调快速传输而非传输顺序
  • 传输的数据可能丢失也可能损毁
  • 传输的数据有数据边界
  • 限制每次传输的数据大小

存在数据边界意味着接收数据的次数和传输次数相同。.
